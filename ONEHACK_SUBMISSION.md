# OneHack Submission Brief

## Project Description / Backstory

OneRealm is a fantasy GameFi economy where players build heroes, take sponsored on-chain missions, earn materials, craft profession-specific gear, and return for asynchronous expeditions.

The project is designed around one core product idea:

> onboarding should feel Web2-simple while ownership, progression, and economy stay on-chain.

## Track

`GameFi`

## Why This Project Fits

OneRealm delivers:
- a playable MVP
- clear gameplay mechanics
- on-chain assets and progression
- a real economy loop instead of a static concept demo

## OneChain Product / Ecosystem Alignment

### Direct runtime alignment
- Move-compatible smart contracts
- PTB-based execution flow
- sponsored transactions
- owned-object game sessions

### Ecosystem-facing alignment
- `OnePredict-ready AI Mentor` in the hero screen
- `OnePlay-ready GameFi UX` through instant play flow and low-friction quest loop
- direct links to `OneChain docs` and `ONEbox` in the app and environment config

## MVP Features

- Google login
- gasless hero mint
- mission families: `Raid`, `Harvest`, `Training`
- contract types: `Standard`, `Bounty`, `Expedition`
- archetypes + professions + rank progression
- materials, salvage, blacksmith crafting, profession-gated recipes
- expedition recovery after page refresh

## Technical Highlights

- native randomness for loot commit
- server-authoritative session model
- allowlisted sponsored transaction policy
- rate-limited gas station flow
- on-chain material and gear economy
- E2E flow for `login -> mint -> quest -> craft -> expedition return`
